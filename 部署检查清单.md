### 步骤1：启动服务器

```bash
double-click: 发布到局域网.bat
```

**预期输出**：
```
╔══════════════════════════════════════════════════════════════╗
║         UE5.5 WebUI 局域网服务器启动成功！                    ║
╚══════════════════════════════════════════════════════════════╝

🌐 访问地址:

   本机访问:
   • http://localhost:8080
   • http://localhost:8080/test.html

   局域网访问（让同事使用这个）:
   • http://192.168.1.100:8080           ← 你的IP
   • http://192.168.1.100:8080/test.html ← 分享这个
```

### 步骤2：检查防火墙

**如果同事无法访问**：

1. **检查防火墙状态**：
   ```bash
   netsh advfirewall show allprofiles state
   ```

2. **添加防火墙规则**：
   ```powershell
   # 以管理员身份运行PowerShell
   New-NetFirewallRule -DisplayName "UE5.5 WebUI" -Direction Inbound -Protocol TCP -LocalPort 8080 -Action Allow
   ```

3. **验证端口**：
   ```bash
   netstat -an | findstr "8080"
   ```

### 步骤3：获取IP地址

**方式1：服务器显示**
- 直接复制服务器输出的局域网地址

**方式2：命令查看**
```bash
ipconfig
```
查找IPv4地址

**方式3：批处理脚本**
- 双击运行 `获取访问地址.bat`

## 🔍 故障排查

### 问题1：同事无法访问

**排查步骤**：
1. [ ] 检查防火墙是否关闭/配置正确
2. [ ] 确认在同一网络（ping IP）
3. [ ] 验证IP地址是否正确
4. [ ] 检查服务器是否运行
5. [ ] 确认端口8080已监听

**解决方案**：
```bash
# 1. 添加防火墙规则
New-NetFirewallRule -DisplayName "UE5.5 WebUI" -Direction Inbound -Protocol TCP -LocalPort 8080 -Action Allow

# 2. 验证网络连通性
ping 192.168.1.100

# 3. 检查端口
netstat -an | findstr "8080"
```

### 问题2：页面样式错乱

**原因分析**：
- [ ] 使用了file://协议
- [ ] 浏览器缓存问题
- [ ] 资源文件未加载

**解决方案**：
```bash
# 1. 清除缓存
Ctrl + F5

# 2. 使用HTTP访问
http://localhost:8080/test.html

# 3. 检查控制台错误
F12 → Console
```

### 问题3：UE通信失败

**排查步骤**：
1. [ ] WebUI插件已启用
2. [ ] Widget Blueprint已配置
3. [ ] OnInterfaceEvent已绑定
4. [ ] 事件名称匹配
5. [ ] UE正在运行

**解决方案**：
```javascript
// 1. 检查浏览器控制台
console.log('UE接口:', window.ue);

// 2. 验证事件发送
ue4('TestEvent', { data: 'test' });

// 3. 检查UE日志
// 在UE编辑器中查看Output Log
```

### 问题4：服务器启动失败

**原因分析**：
- [ ] 端口8080被占用
- [ ] Node.js未安装
- [ ] 文件路径错误

**解决方案**：
```bash
# 1. 检查端口占用
netstat -an | findstr "8080"

# 2. 修改端口
# 编辑 start-server-lan.js
const port = 8081; // 修改为其他端口

# 3. 重新启动
node start-server-lan.js
```
