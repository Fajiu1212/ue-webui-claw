# UE5.5 WebUI 局域网发布指南

## 快速开始

### 方法一：一键启动（推荐）

1. **双击运行**：
   ```
   发布到局域网.bat
   ```

2. **查看输出信息**：
   服务器会自动显示：
   - ✅ 本机访问地址
   - 🌐 局域网访问地址（让同事使用这个）
   - 💡 防火墙提示

3. **分享地址给同事**：
   复制类似 `http://192.168.1.100:8080/test.html` 的地址

### 方法二：手动启动

1. **打开命令提示符**：
   ```bash
   Win + R → 输入 cmd → 回车
   ```

2. **进入项目目录**：
   ```bash
   cd /d g:\Claw_Project\Claw_WebUI
   ```

3. **启动服务器**：
   ```bash
   node start-server-lan.js
   ```

## 防火墙配置

### Windows 防火墙设置

#### 方式1：自动添加规则（推荐）

以管理员身份运行PowerShell：
```powershell
# 添加防火墙规则，允许端口8080
New-NetFirewallRule -DisplayName "UE5.5 WebUI" -Direction Inbound -Protocol TCP -LocalPort 8080 -Action Allow

# 查看规则
Get-NetFirewallRule -DisplayName "UE5.5 WebUI"
```

#### 方式2：图形界面设置

1. **打开防火墙设置**：
   - 控制面板 → Windows Defender 防火墙 → 高级设置

2. **添加入站规则**：
   - 右键 "入站规则" → 新建规则
   - 规则类型：端口
   - 协议：TCP
   - 特定本地端口：8080
   - 操作：允许连接
   - 配置文件：全部勾选
   - 名称：UE5.5 WebUI

3. **完成**：
   点击完成保存规则

### 检查端口是否开放

```bash
# 在命令提示符中运行
netstat -an | findstr "8080"
```

应该看到类似：
```
TCP    0.0.0.0:8080           0.0.0.0:0              LISTENING
```

## 故障排查

### 问题1：同事无法访问

**检查步骤**：

1. **检查防火墙**：
   ```bash
   # 临时关闭防火墙测试（不推荐长期使用）
   netsh advfirewall set allprofiles state off
   ```

2. **检查端口监听**：
   ```bash
   netstat -an | findstr "8080"
   ```

3. **检查IP地址**：
   ```bash
   ipconfig
   ```

4. **测试连通性**：
   在同事电脑上：
   ```bash
   ping 你的IP地址
   ```

**解决方案**：
- 确保在同一网络（同一路由器）
- 检查防火墙规则是否正确添加
- 确认IP地址是否正确

### 问题2：页面加载慢

**可能原因**：
- 网络带宽不足
- 同时访问人数过多
- 服务器性能限制

**解决方案**：
- 限制同时访问人数
- 优化页面资源（压缩图片、代码）
- 使用生产模式构建：
  ```bash
  npm run build
  ```

### 问题3：UE通信失败

**检查项**：
- 确认在UE编辑器中运行
- WebUI插件已启用
- Widget Blueprint已正确配置

**测试方法**：
1. 在本机浏览器中打开控制台
2. 点击Test按钮
3. 查看控制台日志
4. 检查UE输出日志

## 性能优化

### 1. 启用Gzip压缩

```javascript
// 在start-server-lan.js中添加
import zlib from 'zlib';

// 在响应中添加压缩
const acceptEncoding = req.headers['accept-encoding'] || '';
if (acceptEncoding.includes('gzip')) {
  res.writeHead(200, { 'Content-Encoding': 'gzip', 'Content-Type': contentType });
  res.end(zlib.gzipSync(content));
} else {
  res.writeHead(200, { 'Content-Type': contentType });
  res.end(content);
}
```

### 2. 启用缓存

```javascript
// 添加缓存头
res.writeHead(200, {
  'Content-Type': contentType,
  'Cache-Control': 'public, max-age=3600' // 缓存1小时
});
```

### 3. 限制并发连接

```javascript
// 在服务器启动前添加
server.maxConnections = 20; // 限制20个并发连接
```

## 安全建议

### 1. 临时访问建议

如果只是临时给同事演示：
- 演示结束后关闭服务器
- 删除防火墙规则
- 不要长期开放端口

### 2. 长期部署建议

如果需要长期访问：
- 使用反向代理（Nginx/Apache）
- 添加认证机制
- 使用HTTPS
- 限制访问IP

### 3. 数据安全

- 不要在页面中显示敏感信息
- 通信数据加密
- 限制UE操作权限

## 常见问题

### Q1: 同事访问显示404？

**A**: 检查文件路径是否正确，确保在 `src/pages` 目录下

### Q2: 样式错乱？

**A**: 确保使用HTTP协议访问，不要使用file://

### Q3: 按钮点击没反应？

**A**: 检查浏览器控制台是否有错误，确认UE接口是否正确

### Q4: 如何停止服务器？

**A**: 在命令窗口按 `Ctrl + C`

### Q5: 如何更改端口？

**A**: 编辑 `start-server-lan.js`，修改 `const port = 8080;`
