# 端口修改说明

## 当前端口

**已修改为：8081**

如果8081端口也被占用，可以继续修改为8082、8083等。

## 修改方法

### 方法1：一键修改（推荐）

使用全局替换功能，将所有8081替换为新的端口号（如8082）：

1. 在VS Code中按 `Ctrl + Shift + H` (全局替换)
2. 查找：8081
3. 替换为：8082
4. 点击"全部替换"

### 方法2：手动修改

需要修改以下文件：

#### 1. start-server.js
```javascript
// 第9行
const port = 8081;  // 修改为新的端口号
```

#### 2. start-server-lan.js
```javascript
// 第11行
const port = 8081;  // 修改为新的端口号
```

#### 3. 获取访问地址.bat
```batch
// 第18、21、32、37行
echo   http://localhost:8081/test.html
echo   http://%LOCAL_IP%:8081/test.html
```

### 方法3：使用文本编辑器批量替换

1. 打开项目文件夹
2. 使用Notepad++或其他编辑器的"在文件中替换"功能
3. 查找目标：8081
4. 替换为：8082
5. 文件类型：*.js, *.bat
6. 点击"全部替换"

## 验证端口是否被占用

### Windows
```bash
# 检查端口8081是否被占用
netstat -an | findstr "8081"

# 查看占用端口的进程
netstat -ano | findstr "8081"

# 查看进程详情
tasklist | findstr "PID"
```

### macOS/Linux
```bash
# 检查端口8081是否被占用
lsof -i :8081

# 查看占用端口的进程
sudo lsof -i :8081
```

## 防火墙设置

如果修改了端口，需要更新防火墙规则：

### 删除旧规则（如果存在）
```powershell
Remove-NetFirewallRule -DisplayName "UE5.5 WebUI"
```

### 添加新规则（以8081为例）
```powershell
New-NetFirewallRule -DisplayName "UE5.5 WebUI" -Direction Inbound -Protocol TCP -LocalPort 8081 -Action Allow
```

### 或者手动添加
1. 控制面板 → Windows Defender防火墙 → 高级设置
2. 入站规则 → 新建规则 → 端口 → TCP → 特定本地端口（8081）→ 允许连接

## 常见问题

### Q: 修改端口后无法启动？
**A**: 
1. 检查新端口是否也被占用
2. 确认所有文件都已修改
3. 重启命令行/终端

### Q: 同事无法访问？
**A**:
1. 检查防火墙是否放行新端口
2. 确认IP地址是否正确
3. 检查网络连接

### Q: 如何快速切换端口？
**A**:
1. 备份当前配置
2. 使用批量替换修改所有端口
3. 更新防火墙规则
